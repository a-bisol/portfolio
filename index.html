Search and Rescue Drone

An autonomous scalable system to allow operators to remotely locate a given target using one to multiple DJI drones.

Search and Rescue Drone

The Search and Rescue Drone system is a fully automated and integrated system to allow operators to search for a selected target.

The system consists of three major components:
  • The Base PC
  • The Mobile Device
  • The DJI Drone
The Base PC

The central webserver running on the client's PC allows the primary operator to have full control of the operation. Upon initialization all mobile controllers are paired with the webserver, where the operator assigns each drone a name and a flight level. The parameters of the search (radial search distance and target object) are then entered by the operator. The server calculates the optimal path for each connected drone and relays the path to the corresponding mobile device.

Once operation is underway, the mobile devices relay all relevant images back to the central server, running them through another object detection scan before showing them to the operator to confirm or deny if the image matches the target. If the target is confirmed, a message is sent to all mobile devices, instructing them to path their drones to observe the detected target.

The Mobile Device

The drone RC app is the key part of the system as it allows for constant communication between the central webserver and a connected drone. Upon initialization, the mobile operator initiates a handshake with the central server to ensure communication is established. Once the takeoff signal is sent by the central server, the mobile operator's role is to ensure legal compliance as well as manually land the drone if the situation calls for it.

As the drone follows the pre-determined route, stills of the camera feed are continually sent to the mobile device. These stills are then processed through an object detection algorithm to scan for potential matches to the target. If any potential matches are detected, a crop of the detection is sent back to the webserver for further analysis.

The DJI Drone

For development purposes, the DJI Mini 3 was used, due to its implementation of the DJI Mobile SDKv5 as well as the relatively low cost. Any DJI drone that supports the MSDKv5 is supported by the system with minimal required adjustments.

Technologies Used:
  • Base PC - HTML, CSS, Python, Flask, OpenCV, YOLOv5
  • Mobile Device - Android Studio, Kotlin, Jetpack Compose, TFLite, DJI MSDK

ADDvantage

An Android application to aid school children who struggle with dyscalculia.

ADDvantage

ADDvantage is a teaching aid that aims to assist children who struggle with dyscalculia. Dyscalculia can impact the ability to visualize and perceive mathematical problems such as addition of items, counting change, or reading an analog clock. It can be treated pretty simply, especially if caught early on, with repeated familiarization of numbers to the inflicted person.

ADDvantage helps in this regard by letting users input their grade level, as well as selecting any potential areas they may struggle with specifically. Questions relevant to their grade are then shown and the users are silently marked on their progress. User scores and areas of struggle can then be shown to a teacher or parent to help figure out a more individualized treatment plan.
Technologies Used:
  • Kotlin, Android Studio, Jetpack Compose

dropTemp

An IoT solution to remotely monitor and track temperature and humidity changes.

dropTemp

dropTemp is a multi-part system that allows users to track temperature and humidity changes with easy to deploy drops. The user registers an account on the primary website and uses that site to view any tracked systems. To start using the system the user needs at least one Arduino equipped with a DHT11 sensor and a bluetooth shield, as well as at least one Raspberry Pi to act as a central relay.

One (or more) Arduinos set up with DHT11 sensors and bluetooth paired to the central Pi continually track the temperature and humidity readings of their sensors. The readings are passed to the Pi with a unique device ID and those readings are then compiled into a table. That table is then uploaded to the central server with the device's IDs, the Pi's ID, and the user's ID to allow the user to view their stats from any internet connected device.

Technologies Used:
  • Arduino - DHT11, HC-05
  • Raspberry Pi - Python, bluez
  • Server - Linux, Apache, MySQL, PHP, NodeRed, HTML, CSS, JavaScript

Social Calendar

A full stack calendar solution to save and share events with your friends.

Social Calendar

Social Calendar is a full-stack deployment consisting of a desktop application and backend server that aims to make event planning infinitely easier by allowing users to see their friend's public events and send out e-vites to new events.

The front end is a light-weight Electron applet that shows the users all their events as well as all their friends private events at a glance. Users can also create their own future events and mark them and private or public.

The backend is a FastAPI server that tracks user account creation as well as storing all friend connections and event data. Any interaction with the front end applet also communicates with the server to ensure proper data persistence.

Technologies Used:
  • Front End - Electron (HTML, CSS, JavaScript)
  • Back End - Python 3, FastAPI, PostgreSQL
